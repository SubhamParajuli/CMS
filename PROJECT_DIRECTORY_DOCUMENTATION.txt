4) ACCOUNTS APP
============================================================

Path:
- canteen_management/accounts/

Files:
- admin.py
- apps.py
- models.py
- tests.py
- views.py
- migrations/

Key responsibilities:
- Defines the custom user model.
- Stores user role and 5-digit user code.
- Supports admin, student, and teacher roles.
- Registers `CustomUser` in Django admin.

Important model behavior:
- `CustomUser` extends `AbstractUser`.
- Extra fields:
  - `user_code` (unique 5-digit string)
  - `role` (`admin`, `student`, `teacher`)
- `is_canteen_admin` property treats admin role and superuser as canteen admins.
- Validation enforces valid role and valid 5-digit `user_code`.
- `save()` keeps admin users marked as staff.
- `save()` skips full validation when Django only updates `last_login`.

Migrations:
- `0001_initial.py` creates the custom user model.
- `0002_backfill_roles_and_add_role_constraint.py` repairs blank roles and adds a role constraint.
- `0003_backfill_invalid_user_codes.py` repairs blank or invalid legacy user codes.

Tests cover:
- superuser defaults
- admin role behavior

Models:
- `Order`
  - user
  - order_date
  - total_amount
  - is_paid
- `OrderItem`
  - order
  - item
  - quantity
  - price_at_purchase

Checkout behavior:
- requires login
- requires POST
- expects JSON body with cart data
- validates full cart before committing data
- rejects invalid or negative quantities
- locks inventory rows during validation
- creates order, order items, payment, and receipt atomically
- reduces stock and flips `is_available` to false when quantity reaches zero

Tests cover:
- auth redirect
- negative quantity rejection

- rollback on payment creation failure

- apps.py
- models.py

- templates/receipt.html

Responsibilities:

- Renders the receipt page.
- Provides a payment consistency audit/repair command.

Models:
- `Payment`
  - one-to-one with `Order`
  - payment_method
  - amount_paid
  - payment_time
- `Receipt`
  - one-to-one with `Order`
  - generated_at

View:
- `receipt_view`
  - requires login
  - only returns receipts for the current user
  - requires an already paid order
  - requires existing `Payment` and `Receipt` rows

Management command:
- `payment_consistency`
  - audits missing payments
  - audits missing receipts
  - audits mismatched payment amounts
  - audits unpaid orders that still have payment/receipt rows
  - supports `--repair`
  - supports `--fail-on-issues`

Tests cover:
- strict receipt view behavior
- anonymous receipt redirect
- audit output
- repair behavior
- fail-on-issues mode
